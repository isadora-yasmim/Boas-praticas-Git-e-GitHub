## 🛠️ Boas práticas essenciais

✔️ Commits pequenos e descritivos
✔️ Uso de padrão (Conventional Commits)
✔️ Branches curtas
✔️ Pull Requests claros
✔️ Revisão antes de merge

🚫 Evite:

* `update`
* `fix bug`
* commits gigantes
* trabalhar direto na `main`

## 🤝 Contribua

Quer melhorar esse projeto?

```bash
git checkout -b feat/minha-contribuicao
git commit -m "feat: melhorar documentação"
git push origin feat/minha-contribuicao
```

Depois é só abrir um PR 🚀
